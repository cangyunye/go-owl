# 数据库配置

go-owl 支持两种嵌入式数据库，通过 Go 构建标签在编译时选择：

## 编译选择

### SQLite3（默认）

SQLite3 是一个成熟的轻量级数据库，零 CGo 兼容性问题，默认编译即可使用。

```bash
# 默认编译，使用 SQLite3
go build -o owl ./cmd/cli
```

### DuckDB（备选）

DuckDB 是一个高性能分析型数据库，适合复杂查询和数据分析。需要在编译时指定标签。

```bash
# 使用构建标签编译 DuckDB 版本
go build -tags duckdb -o owl ./cmd/cli
```

注意：DuckDB 依赖 CGo 绑定库，在部分 Linux 环境中可能需要安装额外的依赖。

## 构建标签说明

| 标签 | 数据库 | 默认 | 说明 |
|------|--------|------|------|
| 无 | SQLite3 | ✅ 默认 | 推荐，兼容性最好 |
| `duckdb` | DuckDB | — | 分析型查询更优，但构建较复杂 |

## 数据库文件

- **SQLite3**（默认）: `~/.owl/history.db`
- **DuckDB**: `~/.owl/history.db`

## 差异对比

| 特性 | SQLite3 | DuckDB |
|-----|---------|--------|
| 查询性能 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 兼容性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 二进制体积 | 较小 | 较大 |
| 类型支持 | JSON 存为 TEXT | 原生 JSON |
| 构建时间 | 较快 | 较慢（需编译 CGO） |
| 跨平台 | ✅ 全平台一致 | ⚠️ 依赖平台 C 库 |

## 推荐使用场景

**SQLite3（默认）**:
- 大多数场景
- 需要稳定跨平台编译的环境
- 简单的 CRUD 操作
- 追求最小体积

**DuckDB**:
- 需要复杂历史查询和数据分析
- 数据量较大
- 可以处理 CGo 编译的环境

# AI 助手模块架构文档

## 1. 概述

AI 助手模块提供自然语言交互界面，让用户可以通过对话式操作来使用 owl 的各项功能，包括节点管理、命令执行、剧本执行和文件传输等。

## 2. 核心组件

### 2.1 Agent
- 位置: `internal/ai/agent.go`
- 职责: AI 交互的核心协调器，处理用户输入、调用 LLM、执行工具

### 2.2 Tools
- 位置: `internal/ai/tools.go`
- 职责: 封装各项功能的工具集，供 AI 调用

### 2.3 Prompts
- 位置: `internal/ai/prompts/`
- 职责: 管理各种系统提示词和任务提示词

## 3. 工具实现方式

### 3.1 当前实现（方案 A）

当前 AI 工具采用 **直接调用 owl 子命令**的方式：

```go
// 示例：QueryNodesTool.Execute
func (t *QueryNodesTool) Execute(ctx context.Context, args map[string]interface{}) (string, error) {
    // 构建 owl 子命令参数
    cmdArgs := []string{"node", "list"}
    
    // 执行命令
    output, err := runOwlCommand(ctx, cmdArgs)
    
    // 如果命令执行失败，回退到直接调用公共逻辑
    if err != nil {
        return t.fallbackExecute(ctx, args)
    }
    
    return output, nil
}
```

#### 优点
- **输出完全一致**: AI 返回的输出与直接使用 owl 命令完全一致
- **维护成本低**: 子命令的任何更新都会自动反映在 AI 输出中
- **实现简单**: 不需要重构现有代码，直接调用即可

#### 缺点
- **性能略低**: 需要启动新进程，有一定开销
- **缺少结构化数据**: 只能获取文本输出，无法直接访问结构化数据

### 3.2 未来优化（方案 B）

后续可以优化为 **直接调用公共逻辑**的方式：

#### 优化方案
1. **提取公共函数**: 将各子命令的核心逻辑提取为可复用的公共函数
2. **统一输出格式化**: 建立统一的输出格式化函数库
3. **工具直接调用**: AI 工具直接调用这些公共函数，避免进程启动开销

#### 优点
- **性能更好**: 避免进程启动开销
- **更好的控制**: 可以直接访问结构化数据
- **更灵活的处理**: 可以在函数层面进行更复杂的处理

#### 实施建议
- 先在性能敏感的工具上试点（如 `node list`）
- 确保输出格式化逻辑与现有子命令完全一致
- 保留方案 A 作为 fallback，确保兼容性

## 4. 日志系统

### 4.1 日志分级
- **Debug**: 详细的调试信息（默认隐藏）
- **Info**: 一般信息（默认隐藏）
- **Warn**: 警告信息
- **Error**: 错误信息

### 4.2 控制方式
- 命令行参数: `--verbose` 或 `-v`
- 别名: `--debug` 保留以兼容旧版本

### 4.3 使用场景
- **用户日常使用**: 默认简洁模式，只显示警告和错误
- **开发调试**: 使用 `--verbose` 查看完整交互过程

## 5. 路由系统

### 5.1 路由流程
1. 用户输入 → 路由 LLM → 判断任务类型
2. 根据任务类型选择对应提示词
3. 调用对应工具执行

### 5.2 支持的路由
- `node`: 节点管理
- `exec_run`: 执行命令
- `exec_script`: 执行脚本
- `file`: 文件传输
- `playbook_list`: 列出剧本
- `playbook_run`: 执行剧本
- `playbook_info`: 查看剧本详情
- `playbook_validate`: 验证剧本

## 6. 测试策略

### 6.1 测试环境隔离
- 使用 `DisableRealCommands` 变量禁用真实命令调用
- 工具会自动回退到模拟实现，确保测试不依赖外部环境

### 6.2 测试覆盖
- 工具调用验证
- 路由逻辑测试
- 提示词解析测试
- 参数解析测试

## 7. 未来优化方向

1. **性能优化**: 从方案 A 逐步迁移到方案 B，直接调用公共逻辑
2. **更多工具**: 增加更多 owl 功能的 AI 工具支持
3. **更智能的路由**: 提升路由 LLM 的判断准确性
4. **上下文记忆**: 更好的会话上下文管理
5. **多轮对话优化**: 优化多轮对话的工具调用流程
